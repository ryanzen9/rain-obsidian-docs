# CLAUDE.md

## CLAUDE.md File

CLAUDE.md 为 Claude 提供持久化上下文内容，用于：编码规范、工作流、项目架构。  
CLAUDE.md 内容是在系统提示符之后作为用户消息传递的，因此不一定百分百执行。

- 存储位置：`./CLAUDE.md` 或 `./.claude/CLAUDE.md`。

运行 `/init` 可自动生成起始 CLAUDE.md 文件。Claude 分析您的代码库并创建一个包含其发现的构建命令、测试指令和项目约定的文件。如果已存在 CLAUDE.md 文件，`/init` 会建议改进而不是覆盖它。然后，您可以根据 Claude 无法自行发现的指令进行完善。

> /init 的 Prompt: I'll analyze this codebase to create a CLAUDE.md file. Let me start by exploring the project structure.

### 编写原则

- **Structure**: 编写指令的方式会影响 Claude 按照指令执行操作的可靠性。用 Markdown 标题和项目符号来组织相关指令。具体、简洁、结构良好的指令效果最佳。
- **Size**: 每次会话加载到上下文的前 200 行中。因此长度最好不大于 200 行。如果 CLAUDE.md 过长，请使用导入或 `.claude/rules/` 文件将其拆分。
- **Specificity**: 编写足够具体的指令以便验证，如：使用 2 空格缩进，而不是“正确格式化代码”。
- **Consistency**: 如果两条规则相互矛盾，Claude 可能会任意选择其中一条。定期检查以删除过时或冲突的指令。

CLAUDE.md 支持使用 `@path/to/import` 进入导入附加文件。注意：附件文件导入后会加载到上下文中。

```markdown
See @README for project overview and @package.json for available npm commands for this project.

# Additional Instructions

- git workflow @docs/git-instructions.md
```

## Rule

对于较大的项目，您可以使用 `.claude/rules/` 目录将指令组织到多个文件中。这使指令模块化，并使团队更容易维护。规则还可以限定在特定的文件路径范围内，以便仅在 Claude 处理匹配文件时加载到上下文中，从而减少干扰并节省上下文空间。

```markdown
your-project/
├── .claude/
│ ├── CLAUDE.md # Main project instructions
│ └── rules/
│ ├── code-style.md # Code style guidelines
│ ├── testing.md # Testing conventions
│ └── security.md # Security requirements
```

使用 yaml 语法将 rule 匹配特定文件，路径限定规则在 Claude 读取符合模式的文件时触发，而不是在每次工具使用时触发。

```markdown
---

paths:

- "src/\*_/_.{ts,tsx}"
- "lib/\*_/_.ts"
- "tests/\*_/_.test.ts"

---

# API Development Rules

- All API endpoints must include input validation
- Use the standard error response format
- Include OpenAPI documentation comments
```

全局的用户 rule 存储在 `~/.claude/rules` 下。

## Example

```markdown
# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

你是一位世界级的首席全栈软件工程师。你擅长使用 NextJs 构建安全的，生产的企业级 Web 应用。

## Project Overview

该项目是一个成熟的，用于生产的企业官网项目。该项目采用 Next.js（App Router）+ Turborepo + SupaBase 的前后端一体 monorepo 架构：以 Next.js 承担页面渲染与接口层（SSR/SSG/ISR + Server Actions/Route Handlers），Turborepo 负责多包组织与构建编排（apps/web + packages 分层复用），SupaBase 作为核心数据存储，配合 Prisma 实现类型安全的数据访问，整体形成兼顾 SEO、性能与可扩展性的生产级企业官网架构。

## Architecture

### Project Base

- Next.js 14
- React 18
- TypeScript
- Tailwind CSS
- shadcn/ui 组件
- Turborepo
- Pnpm Workspace

company-monorepo/

## WorkFLows

使用 EPIV 模式:

**Explore**： Use Tool `AskUserQuestion`,反向采访用户，理清边缘情况、技术实现和权衡。

**Plan**： 进入Plan Mode，创建详细的实施计划，用户进行审阅。

**Implement**：切回普通模式，让 Claude 根据计划进行编码。

**Verification**：验证本次的编码是否符合预期目标，寻找是否出现类型错误，项目是否能照常运行。
```

进入命令行执行 `claude`，输入：`Hi Claude，我想要构建成熟的，用于生产的企业官网项目，请你读取 \`CLAUDE.md\` 文件，帮我初始化这个项目框架。并且帮我书写一个简单的 index，构建并进行验证。`
