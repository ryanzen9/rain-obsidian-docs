# ClaudeCode

## Agent Loop

当你给 Claude 一个任务时，它会经历三个阶段：收集上下文、采取行动和验证结果。这些阶段相互融合。Claude 在整个过程中使用工具，无论是搜索文件以理解你的代码、编辑以进行修改，还是运行测试以检查其工作。

![示例图片](https://mintcdn.com/claude-code/c5r9_6tjPMzFdDDT/images/agentic-loop.svg?w=2500&fit=max&auto=format&n=c5r9_6tjPMzFdDDT&q=85&s=dfee4a0224b22047f2fecdaf8b3eba3e)

这个智能体循环由两个组件驱动：推理模型和行动工具。Claude 根据从上一步学到的知识决定每一步需要什么，将数十个行动串联起来，并在过程中进行修正。你可以在任何时刻中断，引导 Claude 走向不同的方向，提供额外的上下文，或要求它尝试不同的方法。

model：此处只需要知道 Sonnet 能够很好地处理大多数编码任务。Opus 在复杂的架构决策方面提供更强的推理能力。在会话期间使用 /model 进行切换。

Claude 只能以文本形式回应。有了工具，Claude 可以执行操作：读取代码、编辑文件、运行命令、搜索网络以及与外部服务交互。每次工具使用都会返回信息，这些信息会反馈到循环中，为 Claude 的下一步决策提供依据。

tools：

- File operations: 文件操作 读取文件、编辑代码、创建新文件、重命名和重新组织
- Search: 搜索 按模式查找文件、使用正则表达式搜索内容、探索代码库
- Execution: 执行: 运行 shell 命令，启动服务器，运行测试，使用 git
- Web 网络: 搜索网络，获取文档，查找错误消息
- Code intelligence 代码智能: 查看编辑后的类型错误和警告，跳转到定义，查找引用（需要代码智能插件）

## Session Manage

每个 Claude Code 对话都是一个与您当前目录绑定的会话。Claude Code 在你工作时将对话保存在本地。每条消息、工具使用和结果都会被存储，这使得你可以回溯、继续和分支会话。会话是独立的。每个新的会话都以一个全新的上下文窗口开始，不包含先前会话的对话历史。

Claude 的表现会随着上下文窗口的填满而下降。

### Manage context

运行 /context 查看哪些内容正在使用空间

CLAUDE.md 为 Claude 提供持久化上下文，每次会话加入上下文中。Claude Code 会自动管理上下文，当您接近限制时。它会先清除较旧的工具输出，如果需要，再对对话进行总结。您的请求和关键代码片段会被保留；早期对话中的详细指令可能会丢失。请将持久性规则放在 CLAUDE.md 中，而不是依赖对话历史。

> 如果会话太长，使用 /compact 命令让 Claude 总结关键代码模式和决策，释放空间。使用 /btw 可以进行不留痕提问，这样可以避免无关细节占用上下文

使用 Skills 和 SubAgents 控制上下文长度

- Skills 按需加载。Claude 在会话开始时看到技能描述，但完整内容仅在技能被使用时才加载。

- SubAgents 拥有自己独立的全新上下文，与您的主对话完全分离。它们的工作不会使您的上下文膨胀。

让 Skills 来控制 Claude Code 的行为稳定化，分发 SubAgents 来进行独立性强的子任务来控制上下文长度。

## 相关概念

1. CLAUDE.md
   概念：这是一个特殊的 Markdown 文件，提供持久化的上下文，Claude 在每次对话启动时都会默认完整加载它，为了避免消耗过多上下文，建议保持简短（200行以内）。
2. .claude/rules/ (规则)
   概念：类似于 CLAUDE.md，但可以将指令拆分成多个文件，类似匹配文件后缀后将对应的提示词写入上下文中。
3. Skills (技能)
   概念：为 Claude 提供领域知识、可复用的工作流和参考材料。与 CLAUDE.md 不同，它只有在需要时（按需）才会被加载，或通过输入 /skill-name 手动触发。
4. Subagents (子代理)
   概念：在完全独立的上下文中运行的专门助手。它们可以执行任务并仅将总结结果返回给主对话，SubAgents在完成复杂调查的同时，保持主会话的上下文干净整洁。
5. Agent teams (代理团队)
   概念：协调多个独立的 Claude Code 会话，它们可以共享任务清单并相互发送消息通信。
6. MCP (Model Context Protocol / 模型上下文协议)
   概念：连接 Claude 与外部服务、工具和数据的协议接口。例如需要让 Agent 查询数据库，读取 Figma 设计稿。
7. Hooks (钩子)
   概念：在特定事件发生时（如文件编辑后、工具执行时）自动触发的确定性脚本，不涉及大语言模型推理，适用于强制执行的自动化操作，例如 ESLINT。
8. Plugins (插件)
   概念：将上述的 Skills、Hooks、Subagents 和 MCP 服务器打包成一个可安装的统一模块。社区内有封装好的完整包。

## 协作方式

与 Claude Code 协同开发的几个原则：

- 严密管理上下文：及时清空：在进行不相关的任务之间，务必使用 /clear 清空上下文。一个拥有更好提示词的全新会话，几乎总是胜过一个积累了大量修正历史的长会话。对于想要让他记住的上下文应该将其持久化在提示词。/compact 命令让 Claude 总结关键代码模式和决策，压缩上下文空间、
- 提供验证机制：对于一个任务，如果没有明确的成功标准，Claude 就不知道什么是真正的成功。需要给出明确的验证方式。如：不要只说“实现一个邮箱验证函数”，而是说“编写验证函数，运行 user@example.com 应为 true，invalid 应为 false。完成后运行测试验证”.
- 授权而非命令: 把 Claude 当作能力强的同事。给出背景和方向，让 Claude 自己决定去读哪些文件、运行哪些命令，而不是微观管理它的每一步。对于明确想要执行的行动善用 SubAgent 控制上下文，保持你的主会话干净整洁。
- 工作流范式：先探索，后计划，再实施 (Explore, Plan, then Code) 探索 (Explore)：进入计划模式（Plan Mode），让 Claude 阅读文件并回答问题，不修改代码。计划 (Plan)：要求 Claude 创建详细的实施计划，你可以按 Ctrl+G 直接在编辑器中修改它的计划。实施 (Implement)：切回普通模式，让 Claude 根据计划进行编码，并验证结果。
